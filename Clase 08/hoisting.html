Declaración de Variables, Alcance y Hoisting

Esta guía interactiva resume cómo el motor de JavaScript (ES6+) gestiona la memoria y la visibilidad de las variables antes y durante la ejecución del código. Comprender estos conceptos es vital para evitar errores comunes relacionados con el uso de variables antes de su definición y la fuga de datos fuera de los ámbitos deseados.

Fundamentos de JavaScript

Explora los conceptos fundamentales sobre cómo JavaScript lee e interpreta las declaraciones de variables y funciones. Haz clic en cada tarjeta para centrarte en sus detalles.

▪Hoisting de var

JavaScript eleva la declaración de las variables var al inicio de su contexto (global o de función) y las inicializa automáticamente como undefined. Esto permite que el código no falle si se accede a ellas antes de la línea de declaración, aunque su valor no sea el final.

▪Hoisting de let y const

Aunque estas variables también se elevan, el motor de JavaScript prohíbe el acceso a ellas hasta que la ejecución llega a la línea donde se declaran. Intentar usarlas antes resulta en un error de referencia (Temporal Dead Zone).

▪Alcance (Scope) de Bloque

Introducido en ES6, permite que variables declaradas con let o const dentro de cualquier par de llaves {} (como en un if, for o while) solo existan dentro de ese bloque, previniendo la fuga de datos.

▪Inmutabilidad con const

A diferencia de let, const requiere una inicialización inmediata y no permite que el identificador sea reasignado a un nuevo valor. Protege referencias clave en tu aplicación.

▪Elevación de Funciones

Las declaraciones de funciones (function nombre() {}) se elevan completamente, lo que significa que el cuerpo de la función está disponible en memoria incluso antes de ser definida en el archivo. Esto es distinto a las expresiones de función asignadas a variables.
